On unhandled exception
This condition is triggered when an exception is thrown, and it's not within a try-catch clause; in other words, the exception has not been caught by any code. This means your program is crashing, and now's the time to do some fast reactions.
It's very important to emphasise that you cannot code anything that is not immediate. Fastloops are immediate, so they can be used as actions in this event. Processes that don't pause MMF2 until they're done - usually actions that need to trigger a condition on completion - are not suitable as the program must exit forcibly as soon as the last On unhandled exception event has finished executing.
Parameters : None.

Console » On any console command
This condition is triggered when any console command is sent through. This means the debugging user has enabled DebugObject's console, and entered a command into it.
Parameters : None.

Console » On a specific console command
This condition is triggered when a specific console command is sent through. This means the debugging user has enabled DebugObject's console, and entered a specific command into it; the command is identified as what the text before the first ' ' character is.
Parameters: Command prefix (text).

Console » On console Ctrl+C event
This condition is triggered when debugging user has enabled DebugObject's console and pressed Ctrl+C whilst typing into the console. Ctrl+C is a special event that normally means a cancelation of the console process. You can chose to handle this as you want.
Parameters : None.

Console » On console Ctrl+Break event
This condition is triggered when debugging user has enabled DebugObject's console and pressed Ctrl+Break whilst typing into the console. Ctrl+Break is a special event that normally means a cancelation of the console process. You can chose to handle this as you want.
Parameters : None.
